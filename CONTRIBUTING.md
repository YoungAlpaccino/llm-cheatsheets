# Contributing

Thanks for helping make this list better! Contributions are welcome and appreciated.

## What belongs here

A resource belongs in this list if it is **genuinely useful for building with or understanding
LLMs**. Quality over quantity — a shorter, trustworthy list beats an exhaustive dump of links.

Good additions:
- Actively maintained tools and libraries.
- High-signal guides, courses, and explainers.
- Cheatsheet improvements, corrections, and new cheatsheets.

## How to contribute

1. **Fork** the repo and create a branch.
2. **Add your entry** to the most relevant section, keeping the existing format:
   ```markdown
   - **[Name](https://link)** — One honest, specific sentence about what it does and when to use it.
   ```
3. Keep the list **alphabetical within a section** where it already is, or grouped logically.
4. **Open a pull request** with a short note on why the resource is worth adding.

## Style rules

- Descriptions must be **specific and honest** — no marketing copy ("revolutionary", "the best").
- One line per entry.
- Link to the canonical source (official repo or docs), not a blog reposting it.
- Fix the link if a resource moved; remove it if it's abandoned/dead.

## What gets declined

- Low-effort self-promotion (your tutorial with 2 stars, your unmaintained wrapper).
- Paid products with no free/open value to the community.
- Duplicate or near-duplicate entries.

## Cheatsheet contributions

Cheatsheets live in [`/cheatsheets`](./cheatsheets). Keep them **dense and copy-pasteable** —
code examples over prose, tables over paragraphs. Verify any code you add actually runs.
